This website is a work in progress with the goals of promoting in person board games in the DC area and helping people find local events. The code is open source and the website will be run as a noncommercial entity.

Email gulu@createthirdplaces.com if you want to contribute or share suggestions.
Updates in the near future will focus on the following features.
  • Updating the styling and UI of the website.
  • Adding more information about board game events in the DC.
  • Improved ability to search for events and groups.
  • Support for hosting events.
The source code can be seen here

If you are interested in helping in one of these specific areas, email gulu@createthirdplaces.com

  • Styling using CSS.
  • JavaScript for interactive content.
  • Helping raise money to support the costs of the site. The site will be run as a non-commercial entity.
  • Evaluating and implementing technologies for the website. Right now, the frontend site is built with Vite, vanilla JavaScript, and vanilla CSS. However, there are other options for the frontend that might be better. The website will also need a backend and alternatives to Node.js and PostgreSQL should be evaluated.
    • CSS: Right now, the website is using vanilla CSS. Sass, Tailwind, and other CSS tools should be evaluated.
    • Vite: Vite appears to be the most popular new tool for building frontend assets. However, there might be other options.
    • Backend API: The website needs a backend API, and it should be decoupled from the UI as much as possible. There are a variety of programing languages and frameworks that can be used. Languages that would be a good fit for the backend include Elixir, Golang, JavaScript(Node.js), PHP, or Rust.
    • Kubernetes or another open source container orchestration tool. The goal is to deploy and run the website without a dependency on a specific cloud provider.
    • Database creation using PostgreSQL or another open source relational database tool.
    • Terraform or another infrastructure as code tool to create and manage infrastructure.
    • CI/CD using Jenkins or another open source tool.
